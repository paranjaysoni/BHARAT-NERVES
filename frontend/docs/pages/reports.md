# Reports Page

## Page Purpose

The Reports page (`/reports`) is the centralized intelligence and reporting hub for Project Aegis. It provides decision makers with structured access to all operational reports, crisis briefs, trade analysis, and executive summaries generated by the Bharat Nerves Platform.

This page is designed as an executive-grade reporting portal — the highest-fidelity informational surface in the application.

---

## Reporting Workflow

1. **Report Generation** — Reports are authored by named agents or analysts and entered into the platform's mock data layer.
2. **Review Process** — Reports enter `draft` → `review` → `published` lifecycle states.
3. **Priority Assignment** — Each report is assigned a priority: `low`, `medium`, `high`, or `critical`.
4. **Selection & Preview** — Users click any report row in the table to load its full preview panel.
5. **Executive Brief** — The Featured Intelligence Brief card surfaces the most critical summary for rapid decision support.

## Layout Structure

### Top Section
- `PageHeader` — title, description
- `StatusBadge` — "Reporting Center Active"

### KPI Row
Six `MetricCard` components:
- Total Reports (124)
- Critical Reports (9)
- Recent Reports (18)
- AI Briefings (22)
- Trade Reports (31)
- Impact Reports (27)

### Main Two-Column Layout (`xl:grid-cols-[1fr_360px]`)

**Left / Main Column:**
- `ReportCategories` — 8 category cards in a 4-column grid
- `ReportsTable` — interactive table with 22 reports; click-to-select row loads preview
- `ReportPreview` — full executive preview panel with summary, findings, recommendations

**Right / Sidebar (360px):**
- `RecentReports` — timeline of latest report activity
- `ReportStatistics` — 4-metric statistics grid
- `FeaturedBrief` — premium intelligence brief card
- `ScenarioReports` — reports linked to the active scenario

### Bottom Section
Two-column layout (`xl:grid-cols-[0.85fr_1.15fr]`):
- `ReportTimeline` — chronological activity log
- `FutureReportingPipeline` — roadmap of planned capabilities

## Current Limitations

- No PDF generation or export
- No scheduled reports
- No email delivery
- No AI-generated report content
- No backend API — all data is static mock
- No file viewing or download
- No report search or filter
- No pagination on the reports table
- Report preview is local state only — not persisted
