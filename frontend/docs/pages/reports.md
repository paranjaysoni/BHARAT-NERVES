# Reports Dashboard

## Purpose

The Reports Dashboard is the executive intelligence reporting center for Bharat Nerves. It provides a search-first report workspace with report discovery, report reading, quick actions, report categories, and recently accessed intelligence.

The current implementation is a static MVP UI refinement that matches the approved Reports reference image. It does not generate, export, download, schedule, or share real reports.

## Layout Hierarchy

The page is implemented in `frontend/src/app/reports/page.tsx` and follows this structure:

1. Page header: `REPORTS` with `Generate, analyze and share intelligence reports`.
2. Top tab navigation and action buttons.
3. Four-card KPI summary row.
4. Main three-column workspace:
   - Left column: Report Library.
   - Center column: Report Viewer.
   - Right column: Quick Actions, Report Categories, Recently Accessed.

No legacy dashboard or report-management sections are rendered.

## Responsive Behavior

The layout is optimized for:

- 1440x900
- 1512x982
- MacBook screens

Large screens preserve the three-column workspace: report list, report viewer, utility sidebar. Smaller screens collapse through CSS grid behavior while preserving section order.

## Design Decisions

- The page is a report workspace, not a report-management dashboard.
- Search and report discovery are primary.
- The report viewer is the central focus.
- Right-side utilities support the selected report.
- Color usage stays within Bharat Nerves dark navy and semantic accent tokens.
- All cards use existing `surface-card`, border, radius, and typography language.

## Known MVP Limitations

- Static mock data only.
- No backend API calls.
- No real report generation.
- No real downloads or exports.
- No scheduling logic.
- No sharing workflow.
- Search, filters, tabs, and pagination are visual only.
- The selected report is fixed in the MVP reference state.
